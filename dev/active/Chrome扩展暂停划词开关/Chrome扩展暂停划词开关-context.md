# Chrome 扩展暂停划词开关 — Context

> 立项日期：2026-04-29  
> **排期**：当前优先完成并合并 **Chrome 扩展插件内 refresh**（`fea/chrome-ext-session-refresh-tri` → `dev`）；本需求 **立项归档，开发顺延**。

## 背景

- 用户希望在 **不进入 `chrome://extensions` 禁用整个扩展** 的前提下，**临时停止划词解释**（无浮层、不触发解释流）。
- 内部测试（多包/双环境）时，也希望 **逻辑关闭** 比系统层开关更轻。

## 用户价值

- 一键「暂停 / 恢复」，降低打扰；测试时减少操作路径。

## 约束与原则

- **默认开启**：与今日行为一致；老用户无存储键视为「开」。
- **关闭划词时**：`content/index.tsx` 顶层的 **「连接插件」`postMessage` 桥接仍运行**（用户可在网站完成连接、写 token），仅 **不挂载** 划词 UI、**不响应** 解释类快捷键（与产品讨论一致）。
- **Popup + Options** 共用同一开关状态（`chrome.storage`，键名在 plan 中敲定）。

## 相关需求

- `dev/active/Chrome扩展插件内refresh/`（应先收尾）  
- 后续若上 **prod/dev 双构建包**，本开关仍适用；不与「多包」互斥。

## 关键决策（立项）

- 做 **扩展内全局「暂停划词」开关**（非卸扩展），Popup + Options 联动；关时保留连接桥。
