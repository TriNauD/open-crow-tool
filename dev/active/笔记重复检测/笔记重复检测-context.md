# 笔记重复检测 Context

## 背景
用户可能对同一个词多次提问，AI 每次回答不同，导致笔记本积累多条内容不同的同名笔记。

## 决策
- 不做语义匹配（向量搜索太重），只做 inputText 标准化后的精确匹配
- 追问笔记不做检测（parentId 不为空时跳过，语义上是深挖子话题，允许重复）
- 游客态也做检测，对比 localStorage 中的已有笔记

## 风险点
- 搜索 API 用 ilike 模糊匹配，前端需要再做精确对比过滤
- 覆盖操作是非原子的（先 DELETE 再 POST），网络异常可能导致旧笔记丢失但新笔记未写入；当前阶段可接受，后续可改为 UPDATE
