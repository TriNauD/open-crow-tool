# 周报 Cron 运维通知邮件 — 背景与约束

## 问题

运维需要确认 **每周 Cron 是否把周报发给所有目标收件人**，并在 **抓取失败、部分发信失败、AI 降级** 时能收到可读摘要，而不是只依赖 Vercel 日志。

## 流程说明（回溯）

**[Decision]** 本条目文档为 **回溯立项**：实现曾在未走完「阶段 1～2 书面方案 + 用户明确批准」的情况下先行合入，违反 `.cursor/rules/dev-workflow.mdc`。后续同类需求须：**讨论定稿 → 写入 `dev/active` + 必要时分卷 → 用户批准 → 再动代码**。

## 约束

- 运维邮件走现有 `sendMail`（Resend 或 SMTP），与订阅邮件同一通道。
- 通知发送失败 **不得** 导致 Cron 返回 500（避免与「周报已部分发出」混淆）；日志中打 `[weekly-digest] ops notify failed`。
- 多运维收件人时隔 **~600ms** 再发下一封，与订阅扇出一致，贴近 Resend 2 req/s。
- 环境变量见 plan；须同步 `.env.local.example`（已办）。

## 验收口径（产品向）

配置了 `DIGEST_OPS_NOTIFY_EMAILS` 后：每周 Cron **成功跑完发信循环** → 收到一封汇总（成功/失败人数、失败邮箱列表、AI 是否降级等）。**抓取阶段即失败** → 收到「中止」类邮件（若同样配置了该变量）。
