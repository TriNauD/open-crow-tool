# 需求与反馈收件箱（个人随手记）

这里放**还没分拣**的念头：头脑风暴碎片、聊天里突然想到的、用着产品冒出来的「也许该做」。**不必写全**，一行一条即可。

**原则**

- **能立刻分清是 bug 的** → 优先去 **GitHub 开 Issue**（打 `bug`），别把复现只死在微信里。
- **大块产品想法** → 定期抄进 [`dev/active/BRAINSTORM需求池/`](../dev/active/BRAINSTORM需求池/README.md) 的对应 `待办-*.md` 或总表，或新建一行。
- **每周花 10～15 分钟**：把本文件清空或打勾——该进池的进池，该关的关；别堆过月。

**与 [notes/brainstorm/](./brainstorm/)**：历史脑暴草稿（如 `BRAINSTORM-todo.md` 等）若在分拣时仍有价值，**合并进**需求池总表或对应 `待办-*.md`，或迁一条到本文件再过筛；已过期可删或文末标「已迁入 / 已弃」。**不要**让 `brainstorm/` 与 `dev/active/BRAINSTORM需求池/` 长期各写一套排期真相。

详细流程：[BRAINSTORM 需求池 README](../dev/active/BRAINSTORM需求池/README.md)（见其中 **「收件箱 → 分拣（进池前）」**）。
